# 发布库

“发布库”用于将当前项目中的本地Blockly库提交到aily Blockly公有库仓库，供其他用户安装使用。

软件会使用你的GitHub账号创建或更新Pull Request（PR）。提交成功表示系统已经受理，不代表库已经通过审核或完成合并。

## 发布前准备

发布前请确认：

- 已登录aily Blockly；
- 拥有可以正常使用的GitHub账号；
- 本地库已经导入当前项目；
- 库已经在真实硬件上测试通过；
- 网络可以正常访问GitHub；
- 库中不包含密码、令牌、私人地址等敏感信息。

如果本地库还没有导入：

1. 打开用于测试该库的项目；
2. 进入“库管理”；
3. 点击“导入本地库”；
4. 选择库的根目录；
5. 等待导入完成，并确认库已经出现在左侧积木工具箱中。

## 检查库文件

库目录至少需要包含以下文件：

```
your-library/
├── package.json
├── block.json
├── toolbox.json
└── generator.js
```

建议同时提供：

```
your-library/
├── readme.md       // 面向用户的使用说明
├── readme_ai.md    // 面向AI的说明，可选
├── src/            // Arduino/C/C++源码，可选
├── i18n/           // 多语言文件，可选
└── pinmaps/        // 引脚映射文件，可选
```

发布时软件会自动检查：

- `package.json`、`block.json`和`toolbox.json`必须是合法JSON；
- `package.json`必须包含字符串类型的`name`和`version`，且本地包名要与已经导入的库名一致；
- `block.json`顶层必须是非空数组，每个积木必须有唯一的`type`；
- `toolbox.json`中的分类和积木配置必须有效；
- `generator.js`必须存在且没有JavaScript语法错误。

如果存在非空的`src/`目录，软件会在提交时自动打包为`src.7z`。如果库根目录中已经有非空的`src.7z`，则会直接使用该文件。

更完整的库结构和开发要求请参阅“库规范”和“库适配”。

## 完成真实硬件测试

发布表单要求确认“我确认该库已在真实硬件上测试通过”。提交前至少应验证：

- 积木可以正常显示、连接和生成代码；
- 生成的代码可以在目标开发板上编译；
- 固件可以成功烧录；
- 主要功能可以在真实硬件上正常运行；
- `package.json`中声明的兼容开发板与实际测试结果一致。

## 打开发布入口

1. 在左侧积木工具箱中找到已经导入的本地库；
2. 右键该库的分类名称；
3. 选择“发布库”。

“发布库”只会出现在本地库的右键菜单中。如果没有看到该选项，请通过“库管理 > 导入本地库”重新导入。

如果当前没有登录，软件会提示先打开“用户中心”登录。已经登录但没有GitHub PR权限时，点击“去授权”，在浏览器中完成授权后返回软件。

当前流程需要GitHub `repo`权限来创建或更新PR。请保持软件开启，并尽量在5分钟内完成授权；授权完成后，软件会自动继续本次发布。

## 填写发布信息

| 字段 | 是否必填 | 填写要求 |
| --- | --- | --- |
| 库名 | 是 | 必须以`@aily-project/lib-`开头；前缀后的名称只能使用小写字母、数字和短横线，并且必须以字母或数字开头和结尾。例如`@aily-project/lib-dht11`。 |
| 版本号 | 是 | 必须使用纯数字的`主版本.次版本.修订版本`格式，例如`1.0.0`。不要添加`v`、日期或预发布后缀。 |
| 显示名称 | 是 | 用户在库管理器和工具箱中看到的名称，例如“DHT11温湿度传感器”。 |
| 库描述 | 否 | 简要说明支持的硬件、主要功能和适用场景。 |
| PR描述 | 否 | 说明本次发布的功能、必要性和测试情况，最多2000个字符。建议认真填写，方便管理员审核。 |
| 作者 | 否 | 填写个人、团队或原库作者名称。使用第三方开源库时，还应在`readme.md`中保留来源和许可证说明。 |
| 关键词 | 否 | 使用英文逗号分隔，例如`sensor, temperature, humidity`。 |

PR描述可以参考以下格式：

```
主要功能：
- （填写本次发布提供的功能）

适用硬件和场景：
- （填写支持的开发板、传感器或模块）

发布原因：
- （说明为什么需要发布该库）

测试情况：
- aily Blockly版本：
- 开发板及核心版本：
- 已验证功能：
- 测试结果：
```

## 是否同步本地信息

发布表单提供“同步保存库名、版本号、显示名称、描述、作者和关键词到本地库package.json”选项。该选项默认不勾选。

- 不勾选：表单内容只用于本次提交，不会修改本地库的`package.json`；
- 勾选：提交成功后，表单内容会保存到本地库；
- 如果修改了库名，软件还会重命名本地库目录，并更新当前项目中的依赖记录。

建议在表单信息已经确认无误时勾选，避免本地信息与发布信息不一致。

如果软件提示重新加载项目，请选择“重新加载”。否则工具箱和依赖信息可能仍然显示旧内容。

## 提交发布

1. 检查库名、版本号和PR描述；
2. 勾选“我确认该库已在真实硬件上测试通过”；
3. 决定是否同步保存到本地`package.json`；
4. 点击“发布库”；
5. 等待“提交已受理”的提示，不要重复点击发布按钮。

提交内容包括库的配置、积木定义、代码生成器、说明文档、多语言和引脚映射文件，以及存在时的源码压缩包。

## 查看提交状态

提交受理后，系统会自动创建或更新PR。可以点击成功提示中的“用户中心”，或稍后进入用户中心的库提交记录页面查看状态。

后续通常包括自动检查和管理员审核。请留意GitHub通知及账号邮箱：

- 审核通过：库会合并到[aily-blockly-libraries](https://github.com/ailyProject/aily-blockly-libraries)公有仓库；
- 需要修改：根据PR中的审核意见修改本地库，再使用同一个库名重新发布；
- 审核未通过：阅读具体原因，修复后再次提交。

软件提示管理员通常会在48小时内审核，实际时间可能受提交量、节假日和修改轮次影响。

## 更新已经提交的库

再次发布同一个库名时，软件会提示“该库已经提交过”：

- 内容没有变化：继续提交不会重复创建新的PR；
- 内容有变化：继续提交会更新已有提交记录，并由后台重新创建或更新PR；
- 库名被其他用户占用：不能覆盖对方的库，请修改为未被占用的名称后再发布。

发布新版本前，应先更新版本号，并在PR描述中说明新增功能、问题修复、兼容范围和测试环境。

## 常见问题

### 右键菜单中没有“发布库”

该入口只对本地库显示。请通过“库管理 > 导入本地库”重新导入，并确认库已经成功加载到左侧工具箱。

### 提示“库信息读取失败”或文件不合规

常见原因包括：

- 缺少`package.json`、`block.json`、`toolbox.json`或`generator.js`；
- JSON文件包含注释、多余逗号或括号不匹配；
- `package.json`中的`name`与当前导入的库名不一致；
- `block.json`为空、积木缺少`type`或多个积木使用相同的`type`；
- `toolbox.json`中的分类缺少`name`或`contents`；
- `generator.js`为空或存在语法错误。

修复后重新加载项目，再次发布。

### 提示库名已经存在或被占用

如果这是你以前提交的库，可以确认后继续更新；如果属于其他用户，请修改库名。不要只修改文件夹名称，应同时修改`package.json`中的`name`。也可以在发布表单中修改库名并勾选同步保存，让软件在提交成功后同步更新本地库和项目依赖。

### GitHub授权后没有继续提交

确认浏览器已经完成授权并成功返回aily Blockly。如果等待超过5分钟、授权被取消、GitHub凭证失效或`repo`权限不足，请重新选择“发布库”并按提示再次授权。

### `src.7z`打包失败

确认`src/`是可以读取的非空目录，并且其中没有被其他程序独占的文件。如果手动提供`src.7z`，请确认它不是空文件，并且压缩包内以`src/`作为顶层目录。

### 已同步本地信息，但工具箱仍显示旧名称

选择提示框中的“重新加载”。如果当时选择了“稍后”，保存项目并重新打开即可生效。

## 提交前检查清单

- [ ] 当前库已经通过“导入本地库”加入项目；
- [ ] 四个必需文件存在且可以正常加载；
- [ ] 库名符合`@aily-project/lib-xxx`规范且未被其他用户占用；
- [ ] 版本号使用`x.y.z`格式并已按本次变更更新；
- [ ] `readme.md`说明了用法、接线、依赖和第三方许可证；
- [ ] 目标开发板编译、烧录和真实硬件运行均已通过；
- [ ] 库内没有账号、密钥或其他敏感信息；
- [ ] PR描述写明了主要功能、适用场景和测试环境；
- [ ] 已决定是否把表单信息同步回本地`package.json`。

更多开发和提交规范请参考[aily-blockly-libraries库规范](https://github.com/ailyProject/aily-blockly-libraries/blob/main/%E5%BA%93%E8%A7%84%E8%8C%83.md)与[贡献指南](https://github.com/ailyProject/aily-blockly-libraries/blob/main/CONTRIBUTING.md)。
