# AI功能使用指南  

在主介面右上為工具列，點擊其中的 ![](../img/0102/ai-btn.webp) 按鈕，即可開始AI互動功能。  

## 模式切換  

點擊AI介面下方的模式切換按鈕，可以切換AI模式  
 ![](../img/0102/mode-btn.webp)  

`問答模式`下，AI具備專案可讀權限，可以收集工作區資訊，給使用者提供問題解答；
`代理模式`下，AI具備對專案的讀寫權限，可調用預設的工具及服務對專案進行操作，但會大量消耗token。

## 常見使用方法

切換到代理模式，可以進行專案分析、自動編程、錯誤修復、轉換庫等多種功能。

### 專案分析
你可以嘗試輸入你的專案需求，可以是模糊的需求：
```
我要開發一個環境資料記錄器，幫我設計。
```
也可以是明確的需求：
```
用ESP32結合DHT22採集溫濕度資料，並儲存到TF卡上。每60秒採集一次，並以系統時間+資料的形式記錄到檔案中，格式為「時間戳，溫度，濕度\n」，如:「123456,32,45\n」
```

### 自動編程
直接描述你要的程式，然後等待AI為你編寫
```
幫我寫一個串口測試程式：使用Arduino UNO R4 minima；115200波特率通訊；每500ms輸出一組數值；每組8個隨機數值，數值間用逗號分隔，單個數值範圍在0~255區間內；每組以換行結尾；每組對應位的數值應該具有連續性，不能波動較大。
```

### 錯誤修復
當編譯、上傳報錯時，點擊查看詳情，可以在日誌中看到報錯內容，然後雙擊報錯內容，可以將錯誤提交到AI輸入框，發送報錯資訊，AI會給出處理建議，或直接幫您處理。

### 轉換庫
如果您有一個arduino、micropython或其他類似軟體的庫，要在本軟體中使用，可以透過AI將庫轉換為本軟體的庫，提示詞如下：
```
「C:\Users\coloz\Downloads\xxx-lib」是arduino xxx庫原始碼，請根據規範，將這個庫轉換為aily blockly庫
```
然後等待庫轉換完成，並安裝即可。

## AI功能配置  
透過AI ASSISTANT介面右上角的設定按鈕，可以打開AI助手設定頁面  
### 使用外部模型  
在設定頁面，你可以填入自有LLM apikey，來調用更多模型。理論支援openAI介面風格所有模型。
### 循環次數
循環次數，是單次agent操作的請求最大次數，最大100次，通常不需要修改。
