# 發布庫

「發布庫」用於將目前專案中的本地Blockly庫提交到aily Blockly公開庫倉庫，供其他使用者安裝及使用。

軟體會使用你的GitHub帳號建立或更新Pull Request（PR）。提交成功只代表系統已受理，不代表庫已通過審核或完成合併。

## 發布前準備

發布前請確認：

- 已登入aily Blockly；
- 擁有可正常使用的GitHub帳號；
- 本地庫已導入目前專案；
- 庫已在真實硬件上測試通過；
- 網路可以正常存取GitHub；
- 庫中不包含密碼、權杖、私人地址等敏感資料。

如果本地庫尚未導入：

1. 開啟用於測試該庫的專案；
2. 進入「庫管理」；
3. 點擊「導入本地庫」；
4. 選擇庫的根目錄；
5. 等待導入完成，並確認庫已出現在左側積木工具箱中。

## 檢查庫檔案

庫目錄至少需要包含以下檔案：

```
your-library/
├── package.json
├── block.json
├── toolbox.json
└── generator.js
```

建議同時提供：

```
your-library/
├── readme.md       // 面向使用者的使用說明
├── readme_ai.md    // 面向AI的說明，可選
├── src/            // Arduino/C/C++原始碼，可選
├── i18n/           // 多語言檔案，可選
└── pinmaps/        // 腳位對應檔案，可選
```

發布時軟體會自動檢查：

- `package.json`、`block.json`及`toolbox.json`必須是合法JSON；
- `package.json`必須包含字串類型的`name`及`version`，而且本地套件名稱要與已導入的庫名一致；
- `block.json`頂層必須是非空陣列，每個積木必須有唯一的`type`；
- `toolbox.json`中的分類及積木設定必須有效；
- `generator.js`必須存在，而且沒有JavaScript語法錯誤。

如果存在非空的`src/`目錄，軟體會在提交時自動封裝為`src.7z`。如果庫根目錄中已有非空的`src.7z`，則會直接使用該檔案。

更完整的庫結構及開發要求請參閱「庫規範」和「庫適配」。

## 完成真實硬件測試

發布表單要求確認「我確認該庫已在真實硬件上測試通過」。提交前至少應驗證：

- 積木可以正常顯示、連接及產生程式碼；
- 產生的程式碼可以在目標開發板上編譯；
- 韌體可以成功燒錄；
- 主要功能可以在真實硬件上正常運作；
- `package.json`中聲明的相容開發板與實際測試結果一致。

## 開啟發布入口

1. 在左側積木工具箱中找到已導入的本地庫；
2. 右鍵點擊該庫的分類名稱；
3. 選擇「發布庫」。

「發布庫」只會出現在本地庫的右鍵選單中。如果沒有看到該選項，請透過「庫管理 > 導入本地庫」重新導入。

如果目前尚未登入，軟體會提示先開啟「使用者中心」登入。已登入但沒有GitHub PR權限時，點擊「前往授權」，在瀏覽器中完成授權後返回軟體。

目前流程需要GitHub `repo`權限來建立或更新PR。請保持軟體開啟，並盡量在5分鐘內完成授權；授權完成後，軟體會自動繼續本次發布。

## 填寫發布資料

| 欄位 | 是否必填 | 填寫要求 |
| --- | --- | --- |
| 庫名 | 是 | 必須以`@aily-project/lib-`開頭；前綴後的名稱只能使用小寫字母、數字及短橫線，而且必須以字母或數字開頭和結尾。例如`@aily-project/lib-dht11`。 |
| 版本號 | 是 | 必須使用純數字的`主版本.次版本.修訂版本`格式，例如`1.0.0`。不要加入`v`、日期或預發布後綴。 |
| 顯示名稱 | 是 | 使用者在庫管理器及工具箱中看到的名稱，例如「DHT11溫濕度感測器」。 |
| 庫描述 | 否 | 簡要說明支援的硬件、主要功能及適用場景。 |
| PR描述 | 否 | 說明本次發布的功能、必要性及測試情況，最多2000個字元。建議認真填寫，方便管理員審核。 |
| 作者 | 否 | 填寫個人、團隊或原庫作者名稱。使用第三方開源庫時，還應在`readme.md`中保留來源及授權條款說明。 |
| 關鍵詞 | 否 | 使用英文逗號分隔，例如`sensor, temperature, humidity`。 |

PR描述可以參考以下格式：

```
主要功能：
- （填寫本次發布提供的功能）

適用硬件及場景：
- （填寫支援的開發板、感測器或模組）

發布原因：
- （說明為何需要發布該庫）

測試情況：
- aily Blockly版本：
- 開發板及核心版本：
- 已驗證功能：
- 測試結果：
```

## 是否同步本地資料

發布表單提供「同步保存庫名、版本號、顯示名稱、描述、作者和關鍵詞到本地庫 package.json」選項。該選項預設不勾選。

- 不勾選：表單內容只用於本次提交，不會修改本地庫的`package.json`；
- 勾選：提交成功後，表單內容會儲存到本地庫；
- 如果修改了庫名，軟體還會重新命名本地庫目錄，並更新目前專案中的相依記錄。

建議在表單資料已確認無誤時勾選，避免本地資料與發布資料不一致。

如果軟體提示重新載入專案，請選擇「重新載入」。否則工具箱及相依資料可能仍然顯示舊內容。

## 提交發布

1. 檢查庫名、版本號及PR描述；
2. 勾選「我確認該庫已在真實硬件上測試通過」；
3. 決定是否同步保存到本地`package.json`；
4. 點擊「發布庫」；
5. 等待「提交已受理」提示，不要重複點擊發布按鈕。

提交內容包括庫設定、積木定義、程式碼產生器、說明文件、多語言及腳位對應檔案，以及存在時的原始碼壓縮檔。

## 查看提交狀態

提交受理後，系統會自動建立或更新PR。可以點擊成功提示中的「使用者中心」，或稍後進入使用者中心的庫提交記錄頁面查看狀態。

後續通常包括自動檢查及管理員審核。請留意GitHub通知及帳號電子郵件：

- 審核通過：庫會合併到[aily-blockly-libraries](https://github.com/ailyProject/aily-blockly-libraries)公開倉庫；
- 需要修改：根據PR中的審核意見修改本地庫，再使用同一個庫名重新發布；
- 審核未通過：閱讀具體原因，修正後再次提交。

軟體提示管理員通常會在48小時內審核，實際時間可能受提交量、假期及修改次數影響。

## 更新已提交的庫

再次發布同一個庫名時，軟體會提示「該庫已提交過」：

- 內容沒有變更：繼續提交不會重複建立新的PR；
- 內容有變更：繼續提交會更新已有提交記錄，並由後台重新建立或更新PR；
- 庫名被其他使用者佔用：不能覆蓋對方的庫，請修改為未被佔用的名稱後再發布。

發布新版本前，應先更新版本號，並在PR描述中說明新增功能、問題修正、相容範圍及測試環境。

## 常見問題

### 右鍵選單中沒有「發布庫」

該入口只對本地庫顯示。請透過「庫管理 > 導入本地庫」重新導入，並確認庫已成功載入到左側工具箱。

### 提示「庫資料讀取失敗」或檔案不合規

常見原因包括：

- 缺少`package.json`、`block.json`、`toolbox.json`或`generator.js`；
- JSON檔案包含註解、多餘逗號或括號不匹配；
- `package.json`中的`name`與目前導入的庫名不一致；
- `block.json`為空、積木缺少`type`或多個積木使用相同的`type`；
- `toolbox.json`中的分類缺少`name`或`contents`；
- `generator.js`為空或存在語法錯誤。

修正後重新載入專案，再次發布。

### 提示庫名已存在或被佔用

如果這是你以前提交的庫，可以確認後繼續更新；如果屬於其他使用者，請修改庫名。不要只修改資料夾名稱，應同時修改`package.json`中的`name`。也可以在發布表單中修改庫名並勾選同步保存，讓軟體在提交成功後同步更新本地庫及專案相依項目。

### GitHub授權後沒有繼續提交

確認瀏覽器已完成授權並成功返回aily Blockly。如果等待超過5分鐘、授權被取消、GitHub憑證失效或`repo`權限不足，請重新選擇「發布庫」並按提示再次授權。

### `src.7z`封裝失敗

確認`src/`是可以讀取的非空目錄，並且其中沒有被其他程式獨佔的檔案。如果手動提供`src.7z`，請確認它不是空檔案，而且壓縮檔內以`src/`作為頂層目錄。

### 已同步本地資料，但工具箱仍顯示舊名稱

選擇提示框中的「重新載入」。如果當時選擇了「稍後」，儲存專案並重新開啟即可生效。

## 提交前檢查清單

- [ ] 目前庫已透過「導入本地庫」加入專案；
- [ ] 四個必要檔案存在且可以正常載入；
- [ ] 庫名符合`@aily-project/lib-xxx`規範且未被其他使用者佔用；
- [ ] 版本號使用`x.y.z`格式，並已按本次變更更新；
- [ ] `readme.md`說明了用法、接線、相依項目及第三方授權條款；
- [ ] 目標開發板編譯、燒錄及真實硬件運作均已通過；
- [ ] 庫內沒有帳號、密鑰或其他敏感資料；
- [ ] PR描述寫明主要功能、適用場景及測試環境；
- [ ] 已決定是否將表單資料同步回本地`package.json`。

更多開發及提交規範請參考[aily-blockly-libraries庫規範](https://github.com/ailyProject/aily-blockly-libraries/blob/main/%E5%BA%93%E8%A7%84%E8%8C%83.md)與[貢獻指南](https://github.com/ailyProject/aily-blockly-libraries/blob/main/CONTRIBUTING.md)。
