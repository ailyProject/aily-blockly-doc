# 開發板適配指南

[開發板倉庫](https://github.com/ailyProject/aily-blockly-boards)

## 開發與除錯方法

### 開啟開發者模式
透過 `選單 > 設定 > 開發者模式` 開啟開發者模式。此時 Blockly 工作區左下角的 `庫管理` 按鈕右側會出現 🔁 圖示，用於在修改庫原始碼後手動重新載入庫。

## 基於現有開發板進行適配
如果你的開發板與 Arduino 官方開發板相近，並使用 avr、renesas 或 esp32，則可以在現有配置中找到類似的配置檔，複製後再按需要修改。

### Arduino 相容開發板適配
目前版本支援採用 avr 和 renesas 核心的 Arduino 相容開發板，請參考 `arduino_uno`、`arduino_uno_r4` 目錄。

### esp32適配
本倉庫提供 esp32 相關模板。先確認你的開發板核心型號，再複製對應資料夾並改成你的開發板名稱，然後按需要修改 `package.json`、`board.json` 即可。
`/template` 是專案初始模板。如果你的開發板自帶部分庫，可以把它們加入 `/template/package.json`，這樣使用者在建立新專案後就會自動載入對應的庫。

### STM32適配
[適配參考 STM32F103C8](https://github.com/ailyProject/aily-blockly-boards/tree/main/stm32f103c8)

### 樹莓派 RP 系列 MCU 適配
[適配參考 Raspberry Pi Pico 2 W](https://github.com/ailyProject/aily-blockly-boards/tree/main/raspberrypi_pico2_w)

### 編譯參數自訂適配
開發板的編譯參數在模板 `board.json` 的 `compilerParam` 參數中設定。如果有特殊編譯需求，可參考以下方式設定，語法與 `arduino-cli` 相同。
```
compile -b esp32:esp32:esp32c3 --build-property build.flash_mode=dio --build-property build.flash_freq=80m --build-property build.flash_size=4MB
compile -b arduino:avr:uno --build-property build.extra_flags=\"-DMY_DEFINE=\"hello world\"\""
compile -b arduino:avr:uno --build-property build.extra_flags=-DPIN=2 \"-DMY_DEFINE=\"hello world\"\"" /home/user/Arduino/MySketch` + "\n" +
compile -b arduino:avr:uno --build-property build.extra_flags=-DPIN=2 --build-property "compiler.cpp.extra_flags=\"-DSSID=\"hello world\"\"" /home/user/Arduino/MySketch` + "\n",
```

### 上傳參數自訂適配

```
avrdude -C${'avrdude.conf'} -v -V -patmega328p -carduino -P${serial} -b${baud} -D -Uflash:w:${file}:i;

esptool --chip esp32s3 --port ${serial} --baud ${baud} --before default_reset --after hard_reset write_flash -z --flash_mode keep --flash_freq keep --flash_size keep 0x0 ${bootloader} 0x8000 ${partitions} 0xe000 ${boot_app0} 0x10000 ${file}

bossac -d --port=${serial} -a -U -e -w ${file} -R
[--use_1200bps_touch, --wait_for_upload] bossac -d --port=${serial} -a -U -e -w ${file} -R

dfu-util --device 0x2341:0x0069,:0x0369 -D ${file} -a0 -Q
```

可替換變數：

- `${serial}`
- `${baud}`
- `${file}`
- `${bootloader}`
- `${partitions}`
- `${boot_app0}`

字串常量：

- `${"檔案名"}`

需要使用 `--use_1200bps_touch` 和 `--wait_for_upload` 時：以 `[]` 包起來並放在最前面。

### 安裝庫
你可以先把庫模板或自己建立的庫放到電腦任意位置，例如 `d:\arduino_test`。
然後在 aily blockly 中開啟專案，再開啟終端並輸入：
```
npm i d:\arduino_test
```
即可為該專案安裝此開發板。

### 重新載入庫
點擊 `專案管理` 右側的 🔁 圖示，即可重新載入專案。

### 修改庫
你可以直接修改專案目錄下的 `node_modules\@aily-project\<開發板名稱>`，然後在 aily blockly 中重新載入測試。
也可以修改 `d:\arduino_test` 下的原始開發板檔案，之後再次安裝庫進行測試。

## 基於新晶片進行適配
如果要為開發板倉庫中尚未適配的晶片新增開發板，可能需要上傳對應的編譯器與工具。由於這些編譯器和工具通常是二進位可執行程式，我們無法判斷其安全性。因此，如需適配全新的晶片，請聯絡官方團隊並提供相關工具的原始碼與可信來源，以避免潛在安全風險。

## 提交庫
你可以先 fork 本專案到個人倉庫，然後把新建的開發板配置放進去，再提交 Pull Request。
聯絡奈何col 合併，或等待其他專案管理員合併。
合併完成後，會推送至 npm 倉庫，此後所有使用者都可以使用你開發的庫。
