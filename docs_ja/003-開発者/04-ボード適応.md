# ボード適応ガイド

[ボードリポジトリ](https://github.com/ailyProject/aily-blockly-boards)

## 開発とデバッグ方法

### 開発者モードを有効にする
`メニュー > 設定 > 開発者モード` から開発者モードを有効にします。すると、Blockly ワークスペース左下の `ライブラリ管理` ボタンの右側に 🔁 アイコンが表示されます。このアイコンは、ライブラリのソースコードを変更した後に手動でライブラリを再読み込みするために使います。

## 既存のボードをベースに適応する
ボードが Arduino 公式ボードと同系統で、avr、renesas、または esp32 を使用している場合は、既存の設定から近い構成ファイルを見つけてコピーし、それをベースに修正できます。

### Arduino互換ボードの適応
現在のバージョンでは、avr および renesas コアの Arduino 互換ボードをサポートしています。`arduino_uno`、`arduino_uno_r4` ディレクトリを参照してください。

### esp32適応
このリポジトリには esp32 関連のテンプレートが用意されています。ボードのコアモデルを確認し、対応するフォルダをコピーしてボード名に変更したうえで、`package.json` と `board.json` を編集してください。
`/template` はプロジェクトの初期テンプレートです。ボードに付属ライブラリがある場合は `/template/package.json` に追加しておくと、ユーザーが新しいプロジェクトを作成した際に対応するライブラリが自動で読み込まれます。

### STM32適応
[参考: STM32F103C8](https://github.com/ailyProject/aily-blockly-boards/tree/main/stm32f103c8)

### コンパイルパラメータのカスタム適応
ボードのコンパイルパラメータは、テンプレート `board.json` ファイル内の `compilerParam` パラメータで設定します。特別なコンパイル設定が必要な場合は、次のような形式で設定できます。使用方法は `arduino-cli` と同じです。
```
compile -b esp32:esp32:esp32c3 --build-property build.flash_mode=dio --build-property build.flash_freq=80m --build-property build.flash_size=4MB
compile -b arduino:avr:uno --build-property build.extra_flags=\"-DMY_DEFINE=\"hello world\"\""
compile -b arduino:avr:uno --build-property build.extra_flags=-DPIN=2 \"-DMY_DEFINE=\"hello world\"\"" /home/user/Arduino/MySketch` + "\n" +
compile -b arduino:avr:uno --build-property build.extra_flags=-DPIN=2 --build-property "compiler.cpp.extra_flags=\"-DSSID=\"hello world\"\"" /home/user/Arduino/MySketch` + "\n",
```

### アップロードパラメータのカスタム適応

```
avrdude -C${'avrdude.conf'} -v -V -patmega328p -carduino -P${serial} -b${baud} -D -Uflash:w:${file}:i;

esptool --chip esp32s3 --port ${serial} --baud ${baud} --before default_reset --after hard_reset write_flash -z --flash_mode keep --flash_freq keep --flash_size keep 0x0 ${bootloader} 0x8000 ${partitions} 0xe000 ${boot_app0} 0x10000 ${file}

bossac -d --port=${serial} -a -U -e -w ${file} -R
[--use_1200bps_touch, --wait_for_upload] bossac -d --port=${serial} -a -U -e -w ${file} -R

dfu-util --device 0x2341:0x0069,:0x0369 -D ${file} -a0 -Q
```

利用可能な変数:

- `${serial}`
- `${baud}`
- `${file}`
- `${bootloader}`
- `${partitions}`
- `${boot_app0}`

文字列定数:

- `${"ファイル名"}`

`--use_1200bps_touch` と `--wait_for_upload` が必要な場合は、`[]` で囲んで先頭に置いてください。

### ライブラリをインストールする
まず、ライブラリテンプレートまたは自作ライブラリを PC 上の任意の場所、たとえば `d:\arduino_test` に配置します。
次に、aily blockly でプロジェクトを開き、ターミナルで次を実行します:
```
npm i d:\arduino_test
```
これでこのプロジェクトにこのボードをインストールできます。

### ライブラリを再読み込みする
`プロジェクト管理` の右側にある 🔁 アイコンをクリックすると、プロジェクトを再読み込みできます。

### ライブラリを変更する
プロジェクトディレクトリ内の `node_modules\@aily-project\<ボード名>` を直接変更し、aily blockly で再読み込みしてテストできます。
または `d:\arduino_test` 内の元のボードファイルを変更し、ライブラリを再インストールしてテストすることもできます。

## 新しいチップをベースに適応する
ボードリポジトリにまだ存在しないチップ向けにボードを追加する場合は、対応するコンパイラやツールをアップロードする必要があることがあります。コンパイラやツールは通常バイナリ実行ファイルであり、安全性をこちらで判断できません。そのため、まったく新しいチップへの適応を行う場合は、潜在的なセキュリティリスクを避けるため、関連ツールのソースコードと信頼できる配布元を添えて公式チームに連絡してください。

## ライブラリを提出する
まずこのプロジェクトを自分のリポジトリにフォークし、新しく作成したボード設定を追加して Pull Request を送信してください。
奈何col にマージを依頼するか、ほかの管理者によるマージを待ってください。
マージ後は npm リポジトリに公開され、すべてのユーザーがあなたの開発したボードライブラリを利用できるようになります。
