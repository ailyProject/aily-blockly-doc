# プログラマーの使用方法
ソフトウェアにはopenocd/probe-rsが統合されており、DAPLink、J-Link、ST-Linkプログラマーをサポートしています。
DAPLinkはそのまま使用できます。J-LinkとST-LinkはZadigを使用してドライバーを置き換えてから使用する必要があります。


### J-Link/ST-Linkドライバー置き換え方法
0. [Zadig](https://zadig.akeo.ie/)をダウンロードします。
1. `管理者権限`でZadigを開きます。
2. メニューバーの Options -> List All Devices をクリックします。
3. ドロップダウンリストに、J-Link/ST-Linkに関連するいくつかの項目が表示されます。慎重に確認してください：

J-Linkの注意事項：
> J-Link (Interface 0) という名前のデバイスを選択してください
> USB IDを確認：VIDは通常1366です

ST-Linkの注意事項：
> ST-Link Debug (Interface 0) という名前のデバイスを選択してください
> USB IDを確認：VIDは通常0483です

4. 右側のドライバー選択ボックスでWinUSBを選択し、Replace Driverをクリックします。
5. 置き換えが完了するまで待ちます。

これでソフトウェアを再度開くと、対応するデバイスが表示され、チップ/開発ボードへの書き込みに使用できるようになります。

### J-Linkドライバーの復元
WinUSBに置き換えた後、Segger公式ソフトウェア（J-Flash、J-Link Configuratorなど）はデバイスを認識できなくなります。公式ツールに戻したい場合は、デバイスマネージャーでデバイスをアンインストールし、「ドライバーの削除」にチェックを入れてから、デバイスを再接続すると元のドライバーが使用できます。

### 既知の問題
STM32F407など、一部の新しいSTM32モデルでは正常に書き込みができない場合があります。これはデバッガー自体のファームウェアにも関係している可能性があります。
